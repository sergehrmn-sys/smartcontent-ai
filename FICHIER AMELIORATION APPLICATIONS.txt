Voici les 7 prompts à copier-coller dans Claude Code, un par un. Attends qu’il termine chaque bloc avant de passer au suivant.

Prompt 1 — Audit rapide du projet avant modification
Analyse rapidement la structure de mon projet sans modifier le code pour l’instant.Objectif :- Identifier le framework utilisé : React, Vite, Next.js ou autre.- Identifier les fichiers principaux de l’interface : App.jsx, App.tsx, Layout.jsx, pages, components, etc.- Identifier où se trouve la sidebar actuelle.- Identifier où se trouve le formulaire du générateur.- Identifier où se trouve la zone des résultats.- Vérifier si Tailwind CSS est déjà installé et configuré.- Vérifier les scripts disponibles dans package.json, surtout dev, build et lint.Ensuite, fais-moi un résumé clair :1. fichiers importants trouvés ;2. composants à modifier ;3. dépendances manquantes ;4. risques éventuels ;5. ordre recommandé des modifications.Ne modifie aucun fichier pendant cette étape.

Prompt 2 — Installer les dépendances et préparer les composants
Maintenant, prépare le projet pour moderniser l’interface.Tâches :1. Installe les dépendances nécessaires :   npm install lucide-react framer-motion2. Vérifie que Tailwind CSS fonctionne correctement.   - Si Tailwind est déjà configuré, ne change rien inutilement.   - Si Tailwind n’est pas configuré, explique-moi d’abord ce qu’il manque avant de modifier.3. Crée un dossier src/components s’il n’existe pas déjà.4. Crée ces fichiers vides ou prépare-les si nécessaire :   - src/components/Sidebar.jsx ou Sidebar.tsx selon le projet   - src/components/StatusBadge.jsx ou StatusBadge.tsx   - src/components/PlatformCard.jsx ou PlatformCard.tsxImportant :- Respecte le langage actuel du projet : si le projet est en JSX, reste en JSX. Si le projet est en TSX, reste en TSX.- Ne casse aucune logique existante.- Ne modifie pas encore le design principal.- À la fin, lance npm run build si disponible et corrige uniquement les erreurs liées à cette étape.

Prompt 3 — Créer une Sidebar SaaS premium
Remplace ou modernise la sidebar actuelle avec un composant Sidebar propre et premium.Objectif visuel :- Interface SaaS moderne.- Fond très sombre : slate-950 ou équivalent.- Bordure droite subtile : border-slate-800.- Espacement plus propre.- Navigation claire.- Icônes avec lucide-react.Contenu de la sidebar :- Logo en haut :  - Icône Sparkles de lucide-react.  - Texte : SmartContent AI.  - Le “AI” peut être en violet.- Bloc statut :  - “Backend en ligne”  - Point vert.  - Fond vert très sombre ou slate avec accent vert.- Informations utilisateur si elles existent déjà :  - Email  - Nom du projet ou workspace  - Plan actuel- Menu :  - Générateur avec icône Sparkles ou WandSparkles.  - Tableau de bord avec icône LayoutDashboard.  - Historique avec icône History.  - Status n8n avec icône Workflow.  - Paramètres avec icône Settings.- Bouton Déconnexion en bas :  - Icône LogOut.  - Style minimaliste.  - hover:bg-slate-800.  - Ne change pas la logique de déconnexion existante.Contraintes :- Ne supprime aucune fonctionnalité existante.- Garde l’élément actif visible avec un fond violet/sombre.- La sidebar doit être responsive :  - desktop : affichée à gauche ;  - mobile/tablette : ne doit pas casser la page. Si nécessaire, la rendre plus compacte.Après modification :- Vérifie que l’app démarre encore.- Lance npm run build si disponible.- Corrige les erreurs éventuelles.

Prompt 4 — Moderniser le layout général et le fond
Modernise le layout principal de l’application.Objectif :Transformer le fond noir plat actuel en fond SaaS moderne avec profondeur visuelle.Tâches :1. Intègre la nouvelle Sidebar à gauche dans le layout principal.2. La zone principale à droite doit utiliser un fond sombre avec gradient radial :   bg-[radial-gradient(ellipse_at_top,_var(--tw-gradient-stops))] from-slate-900 via-slate-950 to-black3. Ajoute une zone de contenu avec :   - max-width confortable ;   - padding propre ;   - espacement vertical régulier ;   - pas de surcharge visuelle.4. En haut à droite du contenu, ajoute un composant StatusBadge :   - texte : “Backend en ligne”   - point vert avec effet animate-ping discret   - fond sombre transparent   - bordure subtile5. Garde le titre principal :   - “Générateur de contenu”   - icône Sparkles ou WandSparkles   - sous-titre : “Décris ton idée, choisis tes plateformes, l’IA génère et adapte automatiquement.”6. Ajoute une très légère décoration visuelle en arrière-plan :   - halo violet/bleu flou   - opacité faible   - ne doit pas gêner la lecture.Contraintes :- Ne change pas la logique métier.- Ne modifie pas les appels API.- Ne modifie pas la génération de contenu.- Ne modifie que le layout et les classes CSS/Tailwind.- Le rendu doit rester lisible sur écran 13 pouces.À la fin :- Vérifie le rendu.- Lance npm run build si disponible.

Prompt 5 — Transformer le formulaire en carte Glassmorphism
Modernise le formulaire du générateur de contenu avec un style glassmorphism premium.Objectif visuel :Le formulaire doit ressembler à une carte SaaS moderne, lumineuse mais professionnelle.Style de la carte :- bg-slate-900/50- backdrop-blur-xl- border border-white/10- rounded-2xl- shadow-2xl- padding généreux- effet hover très subtil si pertinentChamp “Sujet / idée du post” :- Garde la logique existante.- Améliore le textarea :  - bg-slate-950/60  - border border-slate-700  - rounded-xl  - focus:ring-2 focus:ring-violet-500/60  - focus:border-violet-400  - transition  - placeholder propre  - texte bien lisible- Ajoute un compteur de caractères si simple à faire sans casser la logique.Menus déroulants :- Harmonise le style :  - fond sombre  - coins arrondis  - bordure subtile  - focus violetPlateformes :- Améliore les checkboxes :  - style plus premium  - alignement propre  - icône ou badge léger pour les plateformes principales si facile :    LinkedIn, Instagram, Facebook, TikTok, YouTube Shorts.Bouton principal :- Texte : “Générer mon contenu”- Icône : Sparkles ou Rocket- Style :  - gradient from-violet-600 to-indigo-600  - hover:from-violet-500 hover:to-indigo-500  - shadow violet  - rounded-xl  - hauteur plus importante  - texte semi-bold- Utilise framer-motion pour un effet léger au clic :  - whileTap scale 0.98  - whileHover scale 1.01Contraintes :- Ne change pas la logique d’envoi.- Ne change pas les noms des variables importantes.- Ne change pas les appels API.- Ne supprime aucun champ existant.- Si un champ est déjà contrôlé par React, conserve sa gestion actuelle.À la fin :- Lance npm run build si disponible.- Corrige uniquement les erreurs liées à cette étape.

Prompt 6 — Moderniser les résultats avec cartes par plateforme
Modernise la zone des résultats générés.Objectif :Les résultats doivent être plus impressionnants pour une démo, avec une carte séparée par plateforme.Tâches :1. Pour chaque plateforme générée, affiche une carte distincte :   - LinkedIn   - Instagram   - Facebook   - autres plateformes si présentes2. Crée ou utilise un composant PlatformCard.Style de chaque carte :- bg-slate-900/60- border border-white/10- rounded-2xl- shadow-lg- padding propre- hover:border-violet-500/40- transition douceEn haut de chaque carte :- Icône de plateforme ou badge texte.- Nom de la plateforme.- Petit statut : “À l’instant” avec point vert.Contenu :- Texte généré bien lisible.- Hashtags séparés visuellement si possible.- CTA visible.Animation :- Utilise framer-motion.- Chaque carte doit apparaître avec :  - opacity 0 vers 1  - y 12 vers 0  - léger délai entre les cartesEffet typewriter :- Ajoute un effet d’écriture progressive pour le texte généré si cela peut être fait simplement et proprement.- Si c’est trop risqué, crée une version simple et stable sans casser la génération.- L’effet doit être léger, pas trop lent.Bouton “Aperçu visuel” :- Ajoute un bouton dans chaque carte :  - icône Eye  - texte : “Aperçu visuel”- Pour l’instant, ce bouton peut ouvrir ou afficher un aperçu simulé du post dans un style mobile.- Ne branche pas de nouvelle API.Statistiques :- Garde ou améliore les statistiques existantes :  - nombre de mots  - nombre de hashtags  - présence CTA- Présente-les dans des petites cartes horizontales modernes.Contraintes :- Ne casse pas la structure des données reçues de l’API.- Adapte-toi au format actuel des résultats.- Ne modifie pas le backend.- Ne modifie pas n8n.À la fin :- Vérifie que les résultats s’affichent encore après génération.- Lance npm run build si disponible.

Prompt 7 — Responsive, polish final et vérification complète
Fais une passe finale de polish et de vérification responsive.Objectif :L’interface doit être prête pour une démo professionnelle sur ordinateur, tablette et écran plus petit.Tâches responsive :1. Vérifie l’affichage desktop :   - sidebar stable à gauche ;   - contenu bien centré ;   - formulaire lisible ;   - résultats bien espacés.2. Vérifie l’affichage tablette :   - pas de débordement horizontal ;   - cartes de résultats en 1 ou 2 colonnes selon l’espace ;   - formulaire utilisable.3. Vérifie l’affichage mobile :   - sidebar compacte ou adaptée ;   - contenu en une seule colonne ;   - boutons suffisamment grands ;   - texte lisible ;   - aucun élément coupé.Polish visuel :- Uniformise les bordures.- Uniformise les rounded.- Uniformise les espacements.- Uniformise les couleurs violet/indigo/slate.- Corrige les contrastes faibles.- Corrige les alignements.- Corrige les marges trop grandes ou trop petites.- Évite les effets trop agressifs.Accessibilité :- Les boutons doivent avoir des labels clairs.- Les champs doivent avoir des labels visibles.- Les contrastes doivent rester lisibles.- Le focus clavier doit être visible.Vérification technique :1. Lance npm run build si disponible.2. Lance npm run lint si disponible.3. Corrige les erreurs.4. Ne corrige pas de manière massive des fichiers sans rapport.5. Ne modifie pas le backend.6. Ne modifie pas les routes API.7. Ne modifie pas les variables d’environnement.À la fin, donne-moi :- la liste des fichiers modifiés ;- les principales améliorations effectuées ;- les commandes exécutées ;- les erreurs rencontrées et corrigées ;- ce qu’il reste à améliorer plus tard.

Pour avancer sans risque, utilise exactement cet ordre : audit → dépendances → sidebar → layout → formulaire → résultats → responsive.
C’est la façon la plus sûre d’éviter que Claude mélange le design, la logique API et le backend.
